■logicCord csvCheck -- CSVアップロード(チェック)
　▼BLOCK A
	・get now datetime
	・get dbdata
	・read csv file
		→ CSVの行を読み込み時に下記の変換処理を行う。
			→ HLA-A1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-A2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-B1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-B2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-Cw1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-Cw2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-DR1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-DR2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-DQ1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ HLA-DQ2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele A1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele A2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele B1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele B2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele C1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele C2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele DRB1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele DRB2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele DQB1 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ allele DQB2 → 造血幹連携APIを使用して、表示名称を内部コードに変換する。
			→ 保存液量 → 実数値(99.9f)で入力されているので、10倍して整数値に変換する。
			→ 有核細胞数 → 実数値(99.99f)で入力されているので、100倍して整数値に変換する。
			→ CD34+細胞数 → 実数値(99.99f)で入力されているので、100倍して整数値に変換する。
			→ 総コロニー数 → 実数値(99.99f)で入力されているので、100倍して整数値に変換する。
			→ GMコロニー数 → 実数値(99.99f)で入力されているので、100倍して整数値に変換する。
			→ 有核赤血球率 → 実数値(99.99f)で入力されているので、100倍して整数値に変換する。
			→ 採取時有核細胞数 → 実数値(99.99f)で入力されているので、100倍して整数値に変換する。
			→ 採取時液量 → 実数値(9999.9f)で入力されているので、10倍して整数値に変換する。

　▼BLOCK B
	・DB Connect
	・start transaction
	・dbsvc create

	・バンクコード、バンク内管理番号、調整バンクコードの組み合わせで同じ値が存在しない場合
		→ 新規登録とする。
	・バンクコード、バンク内管理番号、調整バンクコードの組み合わせで同じ値が存在する場合
		→ 更新とする。

　▼BLOCK C
	・validate
		→ CSVファイルのカラム数が定義と違う場合はエラー
		→ バンクコードは必須とする。
		→ バンク内管理番号は必須とする。
		→ 調整バンクコードは必須とする。
		→ 登録状態は必須とする。
		→ HLA-A(1)は必須とする。
		→ HLA-B(1)は必須とする。
		→ HLA-DR(1)は必須とする。
		→ 採取年月日は必須とする。
		→ ABO血液型は必須とする。
		→ Rh血液型は必須とする。
		→ 分離方法性別は必須とする。
		→ 凍結方法は必須とする。
		→ 凍害保護液は必須とする。
		→ 保存液量は必須とする。
		→ 保存温度は必須とする。
		→ 有効細胞数は必須とする。
		→ CD34細胞数は必須とする。
		→ CD34測定方法は必須とする。
		→ CFU総数は必須とする。
		→ CFU-GM数は必須とする。
		→ CFU測定方法は必須とする。
		→ CMV-IgM検査結果は必須とする。
		→ CMV-DNA検査結果は必須とする。
	・関連validate
		→ A(2)が入力されている場合はHLA-A(2)は必須とする。
		→ B(2)が入力されている場合はHLA-B(2)は必須とする。
		→ C(1)が入力されている場合はHLA-C(1)は必須とする。
		→ C(2)が入力されている場合はHLA-C(2)は必須とする。
		→ DRB(2)が入力されている場合はHLA-DR(2)は必須とする。
		→ DQB(1)が入力されている場合はHLA-DQ(1)は必須とする。
		→ DQB(2)が入力されている場合はHLA-DQ(2)は必須とする。
		→ HLA-A(1)とA(1)の組み合わせが正しいかをチェックする。
		→ HLA-A(2)とA(2)の組み合わせが正しいかをチェックする。
		→ HLA-B(1)とB(1)の組み合わせが正しいかをチェックする。
		→ HLA-B(2)とB(2)の組み合わせが正しいかをチェックする。
		→ HLA-C(1)とC(1)の組み合わせが正しいかをチェックする。
		→ HLA-C(3)とC(1)の組み合わせが正しいかをチェックする。
		→ HLA-DR(1)とDRB(1)の組み合わせが正しいかをチェックする。
		→ HLA-DR(2)とDRB(2)の組み合わせが正しいかをチェックする。
		→ HLA-DQ(1)とDQB(1)の組み合わせが正しいかをチェックする。
		→ HLA-DQ(2)とDQB(2)の組み合わせが正しいかをチェックする。
		→ 施設コードとTRUMP施設コードの値が違う場合はエラーとする。
		→ 登録状態によって下記をチェックすること。
			→ 新規登録の場合
				→ 登録状態が「0」、「1」以外の場合はエラーとする。
			→ 更新の場合
				→ 登録状態に設定できる値以外の場合はエラーとすること。
				→ 登録状態が「2」
					→ 備考が必須とする。
				→ 登録状態が「8」
					→ 備考が必須とする。
				→ 登録状態が「9」
					→ 供給年月日と供給病院、施設コード、TRUMP施設コードが必須とする。
				→ 登録状態が「10」
					→ 移植年月日と移植実施フラグが必須とする。
						→ 移植実施フラグが「2」の場合
							→ 備考が必須とする。

　▼BLOCK D
	・DB check validate
		→ 更新の場合
			→ 登録状態によって下記をチェックすること。
				→ 登録状態が「1」
					→ DBの登録状態が「0」、「8」、「9」以外の場合はエラー
				→ 登録状態が「2」
					→ DBの登録状態が「1」、「3」、「4」以外の場合はエラー
				→ 登録状態が「3」
					→ DBの登録状態が「4」以外の場合はエラー
				→ 登録状態が「4」
					→ DBの登録状態が「3」以外の場合はエラー
				→ 登録状態が「8」
					→ DBの登録状態が「0」、「1」、「3」、「4」以外の場合はエラー
				→ 登録状態が「9」
					→ DBの登録状態が「3」以外の場合はエラー
				→ 登録状態が「10」
					→ DBの登録状態が「9」以外の場合はエラー

　▼BLOCK E

　▼BLOCK F
